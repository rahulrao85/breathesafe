# BreatheSafe

> **India's sleep-apnea awareness intelligence layer.**
> A district-level public-health platform that joins NFHS-5 health, Census
> demographics, CPCB air quality, and Google Trends awareness signals in
> BigQuery, then uses Vertex AI (Gemini 2.5 Flash) + ADK on Cloud Run to
> answer one question:
>
> *Where in India should sleep-apnea screening camps be launched first?*

Built for the **Google Cloud Gen AI Academy APAC 2026 — Cohort 2** hackathon
(theme: *Unified Data Analytics & Intelligence*).

---

## 1. Why this is not "Sleep IQ with a new name"

| Sleep IQ answered... | BreatheSafe answers... |
|---|---|
| "What is happening to *my* sleep?" | "Where should *India* screen first?" |

- **No** wearable ingestion
- **No** CPAP / personal data
- **No** individual diagnosis
- **No** "your sleep score"

BreatheSafe is a **population-level screening-prioritization** system.
The output is a ranked list of districts, not a recommendation for any
single person.

## 3. Data

| Source | What it provides | Where it lands |
|---|---|---|
| NFHS-5 district health data | Obesity, hypertension, blood-sugar, tobacco proxies | `breathesafe.health_survey` |
| Census 2011 demographics | Population, age 50+ %, male %, urban %, lat/lon | `breathesafe.demographics` |
| CPCB / OpenAQ-style AQI | PM2.5 annual mean + AQI category | `breathesafe.air_quality` |
| Google Trends awareness | Sleep-apnea / snoring / CPAP search interest | `breathesafe.awareness_signals` |
| STOP-BANG model weights | Explainable population risk weights + citations | `breathesafe.screening_model` |
| WHO / ICMR / NHS / AASM passages | Grounded medical context for the agent | In-repo RAG corpus + Cloud Storage |

**Local demo dataset:** 365 districts across 31 states, generated by
`scripts/process_data.py`. NFHS-5 covers 21 states directly; the
remaining 10 are calibrated synthetic rows for demo completeness.
Air-quality coverage is the 50 most populous / most polluted districts.
All gaps are documented in `backend/bq/schema.sql` and the data card in
the SPA footer.

## 4. Risk model

```
risk_score =
    0.25 * hypertension_normalized +   -- NFHS-5
    0.25 * obesity_normalized +         -- NFHS-5
    0.15 * age_50plus_normalized +      -- Census
    0.10 * male_pct_normalized +        -- Census
    0.25 * pm25_normalized             -- CPCB / OpenAQ

awareness_gap = risk_score * (1 - awareness_normalized)
```

- `awareness_normalized` is a weighted blend of Google Trends interest
  in *sleep apnea* (0.5), *snoring* (0.3), *cpap* (0.2).
- Risk category: HIGH / MODERATE / LOW (P75 / P25 percentile thresholds).
- **Awareness desert** = `risk_score > 0.5 AND awareness_normalized < 0.3`.
- Estimated undiagnosed = `population * 0.096 * (1 - awareness_normalized)`,
  anchored to Sharma et al., *Lancet Respiratory Medicine*, 2020.

The model adapts only STOP-BANG factors with reliable population proxies.
Tiredness, observed apnea, and neck circumference are intentionally
omitted — there is no defensible district-level proxy.

## 5. API surface

Local and Cloud Run expose the same FastAPI app.

| Method | Path | Purpose |
|---|---|---|
| GET | `/health` | Cloud Run health probe |
| GET | `/` | Serves the SPA |
| GET | `/api/summary` | National KPIs (districts, high-risk count, top desert) |
| GET | `/api/districts` | Filterable district list (`?state=`, `?risk_category=`, `?awareness_desert=`, `?sort_by=`, `?limit=`) |
| GET | `/api/states` | State-level rollup |
| GET | `/api/district/{state}/{district}` | Single district |
| POST | `/api/ask` | ADK agent chat (Markdown answer + tool trace) |
| POST | `/api/analyze-image` | Multimodal: location/topic extraction + BQ cross-ref |

Local mode reads from the processed CSVs in `backend/data/processed/`.
Production mode reads from BigQuery when `GOOGLE_APPLICATION_CREDENTIALS`
and `BQ_PROJECT_ID` are set.

## 6. The agent (ADK-style)

Three tools, one router, one composer. The router is a deterministic
keyword+state detector today; the same tool interface is consumed by
Vertex AI (Gemini 2.5 Flash) function-calling in production.

```
User question
      |
      v
   route()    -- picks tools by intent
      |
      +--> query_district_risk  (BigQuery / processed CSV)
      +--> lookup_guidelines    (RAG over 10 curated passages)
      +--> analyze_image        (Vertex AI Vision with mock fallback)
      |
      v
 compose_answer()  -- Markdown answer + driving factors + citations
```

The system prompt enforces: no individual diagnosis, always cite
sources, always include the responsible-AI disclaimer, never invent
numbers.

## 10. Responsible AI

BreatheSafe is a **screening-prioritization** tool. It does not
diagnose individuals. Population risk scores are estimates anchored
to peer-reviewed Indian prevalence data and STOP-BANG validation
literature; they do not predict individual OSA.

The disclaimer is rendered:

- in the SPA footer (every page),
- in every agent response (Markdown tail),
- in the deck (dedicated slide).

If you are concerned about your own sleep, please consult a qualified
physician or a certified sleep clinic.
